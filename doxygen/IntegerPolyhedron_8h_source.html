20 /// An integer polyhedron is the set of solutions to a list of affine
21 /// constraints over n integer-valued variables/identifiers. Affine constraints
22 /// can be inequalities or equalities in the form:
23 ///
24 /// Inequality: c_0*x_0 + c_1*x_1 + .... + c_{n-1}*x_{n-1} + c_n >= 0
25 /// Equality : c_0*x_0 + c_1*x_1 + .... + c_{n-1}*x_{n-1} + c_n == 0
26 ///
27 /// where c_0, c_1, ..., c_n are integers.
28 ///
29 /// Such a set corresponds to the set of integer points lying in a convex
30 /// polyhedron. For example, consider the set: (x, y) : (1 <= x <= 7, x = 2y).
31 /// This set contains the points (2, 1), (4, 2), and (6, 3).
32 ///
33 /// The integer-valued variables are distinguished into 3 types of:
34 ///
35 /// Dimension: Ordinary variables over which the set is represented.
36 ///
37 /// Symbol: Symbol variables correspond to fixed but unknown values.
38 /// Mathematically, an integer polyhedron with symbolic variables is like a
39 /// family of integer polyhedra indexed by the symbolic variables.
40 ///
41 /// Local: Local variables correspond to existentially quantified variables. For
42 /// example, consider the set: (x) : (exists q : 1 <= x <= 7, x = 2q). An
43 /// assignment to symbolic and dimension variables is valid if there exists some
44 /// assignment to the local variable `q` satisfying these constraints. For this
45 /// example, the set is equivalent to {2, 4, 6}. Mathematically, existential
46 /// quantification can be thought of as the result of projection. In this
47 /// example, `q` is existentially quantified. This can be thought of as the
48 /// result of projecting out `q` from the previous example, i.e. we obtained {2,
49 /// 4, 6} by projecting out the second dimension from {(2, 1), (4, 2), (6, 2)}.
50 ///
51 class IntegerPolyhedron {
